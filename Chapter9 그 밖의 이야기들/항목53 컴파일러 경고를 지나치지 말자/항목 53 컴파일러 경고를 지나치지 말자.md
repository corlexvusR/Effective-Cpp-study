# 항목 53: 컴파일러 경고를 지나치지 말자 - 작성자: 서영은

---

# 자주하는 실수

가상함수인 B::f를 D::f에서 재정의(redefine, 오버라이드) 하겠다는 의도

```cpp
class B {
public:
	virtual void f() const;
};

class D: public B {
public:
	virtual void f();
};
```

실수 ⇒ B 클래스의 f는 상수(const) 멤버 함수이지만, D 클래스의 f는 const가 없음.

## 컴파일러 경고

```cpp
warning: D::f() hides virtual B::f()
```

의미: B에서 선언된 f가 재선언된 것이 아니라 아예 가려졌음.

그러니까 컴파일이 되더라도 노란색 경고는 넘어가지 말자.

의도와 다르게 작동할 수 있음.

또 다른 문제! ⇒ 어떤 컴파일러는 이 코드에 대해 경고를 안 함

따라서, 컴파일러 경고에 너무 의존해도 안된다. 경고 없다고 문제 없는 코드는 아님.

<aside>

# 🔎 결론

- 컴파일러 경고를 쉽게 지나치지 말자.
- 컴파일러에서 지원하는 최고 경고 수준에도 경고 메시지를 내지 않고 컴파일되는 코드를 만드는 것에 전력을 다 하자.
- 컴파일러 경고에 너무 기대지 말자!
- 컴파일러마다 경고를 내는 부분이 천차만별이기 때문에 다른 컴파일러로 코드를 이식하면서 경고 메시지가 사라질 수도 있다.
</aside>
