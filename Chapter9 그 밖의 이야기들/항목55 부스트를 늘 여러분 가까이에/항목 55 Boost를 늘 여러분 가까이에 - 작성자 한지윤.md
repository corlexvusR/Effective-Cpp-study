## ⚙️ **Boost의 특징**

### 1️⃣ C++ 표준과 밀접한 관계

- Boost의 대부분은 **표준 위원회(ISO C++)와 긴밀히 연계**되어 있음
- 실제로 C++11, C++14 표준에 포함된 기능 상당수가 Boost에서 유래함
    - `shared_ptr`, `regex`, `function`, `bind`, `lambda`, `thread`, `chrono` 등

> ✅ Boost → 검증 → 표준 채택
> 
> 
> Boost는 일종의 **“C++ 기능 실험장”** 역할을 합니다.
> 

---

### 2️⃣ 공개 심사 시스템 (Public Peer Review)

- 모든 Boost 라이브러리는 **공개 평가와 심사 절차**를 거쳐야 함
- 심사 항목:
    - 코드 품질과 구현 우수성
    - 플랫폼 독립성 (다른 OS에서도 동작 가능?)
    - 문서화 및 예제의 완성도
    - 사용자 입장에서의 유용성
- 심사 결과:
    - `Accepted` (승인)
    - `Conditionally Accepted` (조건부 승인)
    - `Rejected` (반려)

> “Discuss, Refine, Resubmit — 만족할 때까지 반복!”
> 
> 
> → 품질을 위해 철저히 다듬는 문화
> 

---

### 3️⃣ **무료이지만 고품질**

- 오픈소스임에도 **상용 수준의 완성도**
- 코드 일관성, 문서화, 테스트 체계 모두 뛰어남
- 누구나 사용·참여·수정 가능

## 🧩 **Boost의 내부 기술 (템플릿 & 고차 프로그래밍)**

- **TMP (Template Meta Programming)**
    - 컴파일 시점에 타입 정보를 조작하는 고급 기법
    - Boost MPL(Meta Programming Library)이 대표적
    - 예시:
        
        ```cpp
        typedef boost::mpl::list<float, double, long double> floats;
        typedef boost::mpl::push_front<floats, int>::type types;
        
        ```
        
        → 타입 리스트를 컴파일 시점에 조작 가능!
        
- **Lambda 라이브러리**
    - C++11 이전에 람다를 흉내내기 위해 만들어진 Boost 기능
        
        ```cpp
        using namespace boost::lambda;
        std::for_each(v.begin(), v.end(), std::cout << _1 * 2 + 10);
        
        ```
